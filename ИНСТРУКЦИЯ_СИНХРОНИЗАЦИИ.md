# Инструкция по синхронизации исправлений на локальный ПК

## Способ 1: Через Git (рекомендуется)

### Если изменения уже закоммичены и запушены:

1. Откройте проект в VS Code на вашем ПК
2. Откройте терминал (Ctrl+` или View → Terminal)
3. Выполните команду:
   ```bash
   git pull origin cursor/are-you-there-3152
   ```
   Или просто:
   ```bash
   git pull
   ```

### Если изменения нужно закоммитить и запушить:

**В удаленном окружении (где были сделаны изменения):**

1. Проверьте статус:
   ```bash
   git status
   ```

2. Добавьте измененные файлы:
   ```bash
   git add supabase_api.py ИСПРАВЛЕНИЕ_РАЗЛОГИНИВАНИЯ.md
   ```

3. Создайте коммит:
   ```bash
   git commit -m "Исправление разлогинивания: добавлены методы в supabase_api.py"
   ```

4. Запушьте изменения:
   ```bash
   git push origin cursor/are-you-there-3152
   ```

**На вашем ПК в VS Code:**

5. Выполните pull:
   ```bash
   git pull
   ```

## Способ 2: Прямое копирование файлов

Если Git не используется или нужно быстро скопировать файлы:

### Через VS Code Remote (если подключены к удаленному серверу):

1. В VS Code откройте файл `supabase_api.py`
2. Скопируйте весь содержимое (Ctrl+A, Ctrl+C)
3. На локальном ПК откройте тот же файл
4. Вставьте содержимое (Ctrl+V)

### Через терминал (SCP/SFTP):

Если у вас есть доступ по SSH:

```bash
scp user@remote:/workspace/supabase_api.py /path/to/local/project/
scp user@remote:/workspace/ИСПРАВЛЕНИЕ_РАЗЛОГИНИВАНИЯ.md /path/to/local/project/
```

## Способ 3: Через VS Code Source Control

1. В VS Code откройте панель Source Control (Ctrl+Shift+G)
2. Если видите изменения - нажмите на кнопку "..." (три точки)
3. Выберите "Pull" или "Pull, Push" → "Pull"

## Проверка успешной синхронизации

После синхронизации проверьте, что файл `supabase_api.py` содержит новые методы:

```bash
grep -n "def kick_active_session" supabase_api.py
grep -n "def check_user_session_status" supabase_api.py
grep -n "def finish_active_session" supabase_api.py
```

Должны увидеть номера строк, где определены эти методы.
